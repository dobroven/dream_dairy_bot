# 🌙 Dream Diary Bot

Telegram-бот для ведения дневника сновидений. Хранит сны в локальной SQLite-базе, изолируя данные по `user_id`.

## Структура проекта

```
dream_diary/
├── bot.py            # Логика бота (handlers, ConversationHandler)
├── db.py             # Слой базы данных (SQLite)
├── ai.py             # OpenRouter: DeepSeek V3 / Qwen 2.5 72B
├── test_db.py        # Тесты для db.py
├── requirements.txt
├── dreams.db         # Создаётся автоматически при первом запуске
└── prompts/
    ├── dream_map.md  # Промпт для поиска паттернов
    └── summary.md    # База знаний по анализу сновидений
```


## Команды бота

| Команда | Описание |
|---|---|
| `/start` | Главное меню с инлайн‑кнопками |
| `/help` | Подробная справка по всем командам |
| `/add` | Записать новый сон (диалог: название → описание) (кнопка "📥 Добавить") |
| `/list` | Список снов с пагинацией (5 за страницу) (кнопка "📄 Список") |
| `/search <слова>` | Поиск по названию и описанию, до 20 результатов (кнопка "🔍 Поиск") |
| `/delete <#N или название>` | Удалить сон по номеру (с учётом текущей страницы) или названию (кнопка "🗑 Удалить") |
| `/edit <#N или название>` | Редактировать сон: название, описание, дату (кнопка "✏️ Редактировать" при просмотре) |
| `/cancel` | Отменить текущий ввод |

## Кнопки

Бот предлагает **инлайн‑кнопки** в главном меню:

- **🏠 Главная** – вернуть главное меню.
- **📥 Добавить** – начать запись сна.
- **📄 Список** – просмотреть список записей.
- **🔍 Поиск** – ввести запрос для поиска.
- **🗑 Удалить** – ввести номер сна для удаления.
- **🗺 Карта снов** – найти повторяющиеся паттерны. После нажатия предлагает выбрать модель: DeepSeek V3 или Qwen 2.5 72B.

Кнопки дублируют текстовые команды, которые остаются резервным способом управления.

После вызова списка появляются кнопки **«Читать N»**, **«Далее ▶»** / **«◀ Назад»** для пагинации, а после просмотра сна — **«✏️ Редактировать»** и **«◀️ Назад к списку»**.


## Как выглядит запись

Каждый сон содержит:

- **Название** — строка от 1 до 120 символов
- **Описание** — от 1 до 2000 символов
- **Дата** — автоматически (сегодня), можно изменить при редактировании

При попытке сохранить пустой заголовок или описание бот попросит повторить ввод. Превышение лимита длины также блокируется.

## Редактирование

Сон можно отредактировать двумя способами:

1. **Из списка:** Список → Читать N → ✏️ Редактировать → изменить название / описание / дату (или `.` чтобы оставить текущее).
2. **Командой:** `/edit #N` или `/edit <название>`.

Дата принимает форматы: `ДД.ММ.ГГГГ`, `ГГГГ-ММ-ДД`. Если отправить `.` — дата остаётся без изменений.

## Обработка ошибок

- Все операции с базой данных обёрнуты в `try/except sqlite3.Error`. При сбое бот логирует ошибку и возвращает пользователю понятное сообщение (сон не сохранён / не найден / удалён).
- Удаление по номеру корректно работает на любой странице списка: бот запоминает текущее смещение (`offset`) и вычисляет индекс в рамках страницы.
- Поиск ограничен 20 результатами; если их ровно 20, пользователь видит подсказку уточнить запрос.

## База данных

Файл `dreams.db` создаётся рядом с `bot.py`. Структура таблицы:

```sql
CREATE TABLE dreams (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER NOT NULL,   -- Telegram user ID
    date        TEXT    NOT NULL,   -- YYYY-MM-DD
    title       TEXT    NOT NULL,
    description TEXT    NOT NULL,
    created_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_dreams_user_date ON dreams (user_id, date DESC);
```

Каждый пользователь видит только свои сны — изоляция по `user_id`.

Для корректного поиска по кириллице используется `str.casefold()` на стороне Python (SQLite `LOWER` не обрабатывает не-ASCII символы).

## Карта снов

Функция **«🗺 Карта снов»** анализирует все сны пользователя и находит повторяющиеся паттерны (ситуации, предметы, ощущения, персонажи). Результат — список паттернов с количеством снов и примерами.

### Как работает

1. При нажатии на кнопку бот предлагает выбрать модель: **DeepSeek V3** или **Qwen 2.5 72B**.
2. Выбранная модель получает: промпт (`dream_map.md`) + базу знаний (`summary.md`) + все сны пользователя.
3. Модель возвращает JSON со списком паттернов, бот форматирует в сообщение.

### Настройка

1. Зарегистрироваться на [OpenRouter](https://openrouter.ai/keys) и получить API-ключ.
2. Добавить в `.env`:
   ```
   OPENROUTER_API_KEY=sk-or-v1-ваш_ключ
   ```
3. По желанию — наполнить `prompts/summary.md` конспектом по анализу сновидений (используется моделями как база знаний).

### Зависимости

Устанавливаются через `pip install -r requirements.txt`:
- `openai` — клиент OpenRouter (совместим с OpenAI API)

## Деплой (опционально)

### Systemd (Linux VPS)

```ini
[Unit]
Description=Dream Diary Telegram Bot
After=network.target

[Service]
WorkingDirectory=/opt/dream_diary
Environment=TELEGRAM_BOT_TOKEN=ваш_токен
Environment=OPENROUTER_API_KEY=sk-or-v1-ваш_ключ
ExecStart=/opt/dream_diary/.venv/bin/python bot.py
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable dream-diary-bot
systemctl start dream-diary-bot
```

### Docker

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "bot.py"]
```
